# Manual Test フェーズ

## subagent実行

このフェーズは**subagentとして実行**される。
**並列フェーズ (parallel_verification)** の一部として実行される。

---

## 目的

自動テストでカバーできない部分を手動で確認する。

## 入力

- タスクディレクトリ: `{workflowDir}`
- 仕様書: `docs/specs/domains/{domain}/{name}.md`
- テストケース: `test-cases.md`

## 手動テスト項目

### 1. 画面表示確認（UI変更がある場合）

- [ ] レイアウトが仕様通りか
- [ ] レスポンシブ対応が正しいか
- [ ] アニメーションが正常か

### 2. 操作確認

- [ ] ユーザー操作が期待通りに動作するか
- [ ] エラー時のフィードバックが適切か
- [ ] ローディング表示が正しいか

### 3. エッジケース

- [ ] 空のデータでの動作
- [ ] 大量データでの動作
- [ ] ネットワークエラー時の動作

### 4. ブラウザ互換性（該当する場合）

- [ ] Chrome
- [ ] Firefox
- [ ] Safari
- [ ] Edge

## 実行手順

1. **テスト環境の準備**
   - 開発サーバーを起動
   - テストデータを準備

2. **テスト実行**
   - 各テスト項目を順番に確認
   - 結果を記録

3. **問題の報告**
   - 発見した問題を記録
   - 修正が必要な場合は対応

## log.md への記録

```markdown
### MANUAL_TEST（手動テスト）

- テスト環境: [環境情報]
- テスト実行者: AI
- テスト結果:
  - 確認項目: X件
  - OK: X件
  - NG: 0件
- 確認した操作:
  - [操作1]: OK
  - [操作2]: OK
- 発見した問題:
  - なし / [問題の説明]
```

## 問題発見時の対応

### 軽微な問題

1. log.md に記録
2. 次のフェーズで修正を検討

### 重大な問題

1. log.md に記録
2. 実装フェーズに戻って修正
3. 再度テストを実行

## 禁止事項

- テストをスキップする
- 問題を記録せずに次に進む

## 完了条件

- [ ] 全ての手動テスト項目を確認した
- [ ] 重大な問題がないことを確認した
- [ ] log.md に結果を記録した
