# requirementsフェーズ

## 目的
機能要件・非機能要件・受け入れ基準を定義する。

## 入力
- 読み込むファイル: `docs/workflows/{taskName}/research.md`

## タスク
1. research.mdの調査結果を読み込む
2. 機能要件を列挙する（FR-XXX形式）
3. 非機能要件を列挙する（NFR-XXX形式）
4. 受け入れ基準を定義する（AC-XXX形式）
5. 用語があれば用語集に追加

## 出力
- 出力ファイル: `docs/workflows/{taskName}/requirements.md`
- フォーマット: Markdown

## 出力テンプレート
```markdown
# 要件定義: {タスク名}

## 1. 背景と目的
{調査結果を踏まえた背景説明}

## 2. 機能要件
### FR-001: {要件名}
{要件の詳細説明}

### FR-002: {要件名}
...

## 3. 非機能要件
### NFR-001: {要件名}
{要件の詳細説明}

## 4. 受け入れ基準
### AC-001: {基準名}
- [ ] {チェック項目1}
- [ ] {チェック項目2}

## 5. 制約事項
{技術的・運用上の制約}

## 6. 用語定義
| 用語 | 定義 |
|------|------|
```

## 品質基準
- 機能要件が明確かつ検証可能
- 非機能要件が定量的に定義されている
- 受け入れ基準が具体的
