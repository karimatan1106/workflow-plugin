# researchフェーズ

## 目的
要件分析・既存コード調査を行い、タスクの理解を深める。

## 入力
- ユーザーからのタスク説明
- 関連する既存コードベース

## タスク
1. タスクの目的と範囲を理解する
2. 関連する既存コードを探索する（Glob, Grep, Readを使用）
3. 技術的な制約や依存関係を特定する
4. 既存の類似実装があれば参考にする

## 出力
- 出力ファイル: `docs/workflows/{taskName}/research.md`
- フォーマット: Markdown

## 出力テンプレート
```markdown
# 調査結果: {タスク名}

## 調査日: {日付}

## 1. タスク概要
{タスクの目的と範囲}

## 2. 関連コード
{発見した関連コードとその役割}

## 3. 技術的考慮事項
{制約、依存関係、注意点}

## 4. 参考実装
{既存の類似実装があれば}

## 5. 次のフェーズへの引き継ぎ
{requirementsフェーズで検討すべき点}
```

## 品質基準
- タスクの目的が明確に理解されている
- 関連コードが網羅的に調査されている
- 技術的リスクが特定されている
