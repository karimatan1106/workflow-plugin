# test_designフェーズ

## 目的
テスト計画とテストケースを設計する。

## 入力
- 読み込むファイル:
  - `docs/workflows/{taskName}/spec.md`
  - `docs/workflows/{taskName}/state-machine.mmd`
  - `docs/workflows/{taskName}/flowchart.mmd`

## タスク
1. テスト対象を特定する
2. テストケースを設計する
3. テストデータを定義する
4. 境界条件・エッジケースを洗い出す

## 出力
- 出力ファイル: `docs/workflows/{taskName}/test-design.md`
- フォーマット: Markdown

## 出力テンプレート
```markdown
# テスト設計: {タスク名}

## 1. テスト方針
{テストの範囲、アプローチ}

## 2. テストケース
### TC-001: {テストケース名}
- **目的**: {テストの目的}
- **前提条件**: {必要な状態}
- **手順**: {実行手順}
- **期待結果**: {期待される結果}

### TC-002: ...

## 3. 境界条件・エッジケース
| ケース | 入力 | 期待結果 |
|--------|------|----------|

## 4. テストデータ
{テストに必要なデータ}

## 5. 検証チェックリスト
- [ ] {検証項目1}
- [ ] {検証項目2}
```

## 品質基準
- 仕様書の要件がテストでカバーされている
- 境界条件が網羅されている
- テストが再現可能
