# threat_modelingフェーズ（parallel_analysis内）

## 目的
セキュリティ脅威の特定と対策を検討する。

## 入力
- 読み込むファイル: `docs/workflows/{taskName}/requirements.md`

## タスク
1. システムの資産を特定する
2. 脅威を列挙する（STRIDE等を参考）
3. リスクを評価する
4. 対策を提案する

## 出力
- 出力ファイル: `docs/workflows/{taskName}/threat-model.md`
- フォーマット: Markdown

## 出力テンプレート
```markdown
# 脅威モデル: {タスク名}

## 1. システム概要
{セキュリティ観点でのシステム説明}

## 2. 資産
| 資産 | 機密性 | 完全性 | 可用性 |
|------|--------|--------|--------|

## 3. 脅威分析
| 脅威 | リスク | 対策 |
|------|--------|------|

## 4. セキュリティ要件
### SR-001: {要件名}
{要件の詳細}

## 5. 結論
{全体的なリスク評価と推奨事項}
```

## 品質基準
- 主要な脅威が網羅されている
- リスク評価が妥当
- 対策が具体的
