# implementationフェーズ

## 目的
テストを通す実装を行う（TDD Green）。

## 入力
- 読み込むファイル:
  - `docs/workflows/{taskName}/spec.md`
  - `docs/workflows/{taskName}/test-design.md`
  - 既存のテストファイル（*.test.ts）

## タスク
1. 仕様書とテスト設計を読み込む
2. テストが通るように実装を行う
3. 既存コードとの整合性を確認
4. 型エラー・lint エラーを解消

## 出力
- ソースコードファイル（*.ts, *.tsx など）
- 必要に応じて設定ファイルの更新

## 実装ガイドライン
1. **テストファースト**: test_implで作成されたテストを通すことを目標にする
2. **最小実装**: 必要最小限の実装に留める（過剰設計を避ける）
3. **既存パターン踏襲**: コードベースの既存パターンに従う
4. **型安全性**: TypeScriptの型を適切に活用する

## 禁止事項
- テストファイルの変更（test_implフェーズで作成済み）
- 仕様にない機能の追加
- 未テストの機能の追加

## 品質基準
- 全てのテストがパスする
- 型エラーがない
- lintエラーがない
- 仕様書の要件を満たしている
