# テスト設計: Clean Architecture ガイダンス

## テスト方針

本機能はドキュメント追加のみのため、以下の検証を行う。

## 検証項目

### 1. CLAUDE.md 検証

| ID | 検証内容 | 期待結果 |
|----|---------|---------|
| T1 | 推奨プロジェクト構造セクション存在 | セクションが追加されている |
| T2 | ディレクトリ構成図 | domain/application/infrastructure/presentation構造が記載 |
| T3 | 依存関係ルール | 矢印図と説明が含まれている |
| T4 | 適用判断ガイダンス | 推奨/不要ケースが明記 |

### 2. SKILL.md 検証

| ID | 検証内容 | 期待結果 |
|----|---------|---------|
| T5 | プロジェクト構造ガイダンスセクション | セクションが追加されている |
| T6 | フェーズ別考慮事項 | requirements/planning/test_designの観点が記載 |
| T7 | 廃止済みサイズ参照削除 | small/medium参照が最新化されている |

### 3. 整合性検証

| ID | 検証内容 | 期待結果 |
|----|---------|---------|
| T8 | CLAUDE.mdとSKILL.mdの整合性 | 両ファイルで矛盾がない |
| T9 | 既存セクションへの影響 | 他のセクションが破損していない |

## テスト実行方法

1. ファイル変更後、grepで必要なセクションの存在を確認
2. 目視でフォーマットと内容を確認
3. MCPサーバーの動作に影響がないことを確認（起動テスト）
